
## ⚙️ شرح تفصيلي لبنات الجهة اليمنى (Right Side Pins)

---

### D23 (GPIO23) — **SPI MOSI / GPIO عام**

- **الوظيفة الأساسية:**
    
    - في واجهة **SPI الافتراضية** بيشتغل كـ **MOSI** (Master Out Slave In).
        
    - ممكن تستخدمه كـ digital I/O عادي لو مش محتاج SPI.
        
- **أمثلة استخدام:**
    
    - إرسال بيانات إلى شريحة خارجية مثل شاشة أو مستشعر SPI.
        
- **كود سريع:**
  
  ```C
pinMode(23, OUTPUT);
digitalWrite(23, HIGH);
```

---

### D22 (GPIO22) — **I2C SCL / GPIO عام**

- **الوظيفة:** الخط الرئيسي للإشارة **SCL** (clock) في بروتوكول **I2C**.
    
- **الاستخدام:** لربط حساسات مثل MPU6050, OLED, BME280… إلخ.
    
- **كود مثال (I2C):**
  
  ```C
#include <Wire.h>
void setup() {
  Wire.begin(); // يستخدم GPIO22 كـ SCL وGPIO21 كـ SDA افتراضياً
  Serial.begin(115200);
}
```

- **معلومة:** تقدر تغيّر البنات الافتراضية بـ 
```C
Wire.begin(SDA, SCL);
```
  
  ---
  
  ### D21 (GPIO21) — **I2C SDA / GPIO عام**

- **الوظيفة:** خط البيانات **SDA** في I2C.
    
- **نقطة مهمة:** دايمًا استخدم مقاومات سحب (pull-up) بين SDA/SCL و 3.3V (قيم 4.7kΩ عادة) لو ما كانت موجودة بالفعل في الحساس.
    
- **مثال:**
```C
 Wire.begin(21, 22);
```

---
### RX2 (GPIO16) — **UART2 RX / GPIO عام**

- **الوظيفة:** استقبال بيانات من الأجهزة التسلسلية (Serial) مثل GPS, Bluetooth module (غير المدمج).
    
- **المنفذ UART2:** تقدر تستخدمه حرًا، مثل:
  
```C
HardwareSerial Serial2(2);
void setup() {
  Serial2.begin(9600, SERIAL_8N1, 16, 17); // RX=16, TX=17
}
```

- **تحذير:** ما توصل 5V UART مباشرة، استخدم level shifter.
  
  ---
  
  ### TX2 (GPIO17) — **UART2 TX / GPIO عام**

- **الوظيفة:** إرسال بيانات في واجهة UART2.
    
- **مثال:** الكود السابق يظهر استخدامه كـ TX.
    
- **نصيحة:** استخدمه مع 16 لتوصيل أي جهاز تسلسلي خارجي.
  
  ---
  
  ### D19 (GPIO19) — **SPI MISO / GPIO عام**

- **الوظيفة:** **MISO** (Master In Slave Out) في واجهة SPI.
    
- **وظيفة إضافية:** ممكن استخدامه كـ I/O عادي.
    
- **مثال بسيط:**
  
```C
pinMode(19, INPUT);
int val = digitalRead(19);
```


---
### D18 (GPIO18) — **SPI SCK / GPIO عام**

- **الوظيفة:** **SCK** (clock) في بروتوكول SPI.
    
- **أمثلة الأجهزة:** شاشات TFT، وحدات SD Card، مستشعرات.
    
- **ملاحظة:** لو بتتعامل مع أكثر من SPI device استخدم chip select منفصل لكل جهاز.
    

---

### D5 (GPIO5) — **SPI SS / Chip Select**

- **الوظيفة:** عادة تُستخدم كبن اختيار الجهاز (Slave Select) في SPI.
    
- **أمثلة:** `digitalWrite(5, LOW);` لتفعيل الجهاز.
    
- **تحذير:** GPIO5 يُعتبر من بنات الـ **boot-sensitive** أحيانًا. لازم يكون **LOW عند الإقلاع** لبعض الإعدادات.  
    استخدمه بحذر وقت التشغيل الأول.
    
    ---
    
    ### D4 (GPIO4) — **GPIO + Touch (T0)**

- **الوظيفة:** I/O عام + مستشعر لمس.
    
- **ميزة إضافية:** غالبًا بيكون متصل بـ LED داخلي في بعض نسخ اللوحات.
    
- **كود لمس بسيط:**
  
```C
  Serial.println(touchRead(4));
```

---
### D2 (GPIO2) — **Boot Mode Sensitive**

- **الوظيفة:** I/O عام ولكن **حرِج أثناء الإقلاع.**
    
    - لازم يكون **LOW** عند محاولة الدخول في وضع البرمجة (bootloader).
        
    - لذلك لا توصل LED أو مقاومات تسحب عالي بدون حساب.
        
- **في اللوحات التجريبية:** عادة يُترك حرًا أو يُستخدم لLED صغير.
    

---

### D15 (GPIO15) — **Boot Mode Sensitive / SPI CS**

- **الوظيفة:** في وضع SPI الافتراضي يُستخدم كـ **CS** (chip select)،  
    لكنه أيضًا **يؤثر في وضع الإقلاع**.
    
- **القاعدة:** لازم يكون **LOW** أثناء الإقلاع الطبيعي.
    
- **نصيحة:** لو بتستخدمه كخرج، اجعل حالته الافتراضية LOW حتى يقلع الـ ESP32 بشكل صحيح.
    

---

### D0 (GPIO0) — **Boot Mode Pin (يُستخدم في تحميل الكود)**

- **الوظيفة:** البن الحرج المستخدم لوضع البرمجة (Flashing).
    
    - **LOW** أثناء إعادة التشغيل = يدخل وضع البرمجة (upload).
        
    - **HIGH** = التشغيل العادي.
        
- **التحذير:** لا تستخدمه لأغراض أخرى إلا لو متأكد أنك مش هتحتاج تبرمج الجهاز عبر USB.
    

---

### D3 (GPIO3) — **UART0 RX (Serial Communication)**

- **الوظيفة:** هذا هو **RX** الرئيسي المستخدم عند رفع الكود من الـ USB.
    
- **ملاحظة:** موصل فعلاً إلى الـ USB-to-Serial chip على اللوحة، فلا تستخدمه مع أجهزة خارجية.
    
- **تحذير:** أي توصيل خارجي عليه ممكن يعطل تحميل الكود.
    

---

### D1 (GPIO1) — **UART0 TX**

- **الوظيفة:** الإرسال التسلسلي الرئيسي (TX).
    
- **ملاحظة:** يُستخدم في رفع الكود أيضًا.
    
- **ممكن تستخدمه للـ debug:** `Serial.println("msg");`
    

---

### 3V3 — **خرج منظم الجهد**

- **الوظيفة:** تقدر تسحب منه جهد **3.3V مستقر** لتغذية حساسات.
    
- **حد الأمان:** لا تسحب أكتر من ~500mA (حسب اللوحة).
    
- **نصيحة:** لو عندك استهلاك أعلى استخدم مصدر 3.3V خارجي.
